### Linear Regression

#### 1) 단순 선형 회귀 
시험 점수를 예측하는것을 예로 들어보자

시험공부를 많이 할수록 점수가 좋게 나온다. 이때 공부시간(x)과 점수(y)는 어떠한 관계가 있다고 할 수 있다.

이때 x 와 y 가 선형 관계에 있을때 x 가 1개이면 단순 선형 회귀 라 한다.



#### 2) 다변수 선형 회귀

시험 점수가 단순히 공부 시간으로만 결정되는것이 아니고 다른 요인이 있을 때 

- 행렬로 표현 가능하다

- 행렬의 곱으로 다변수 회귀를 처리



#### 3) 가설 함수

시험점수와 공부시간의 관계를 알아보기 위하여 가설함수(hypothesis) 를 세울수있다.

- H(x) = W*X + b 로 나타낸단다 

- W(Weight) 는 가중치 b(bias) 편향이라고 한다.



#### 4) 비용(손실) 함수 

가설 함수의 W , b 값을 조절 하여 주어진 데이터에 맞는 값을 찾아내야한다. 

이를 위하여 현재의 W ,b 값이 얼마나 잘못되어 있는지 ? 에 대한 정보가 필요할 것이다.

- 비용 함수(cost function or loss function) 는 다양한 것이 있지만 회귀문제에서는 평균 제곱 오차? 함수를 쓴다고 한다.
- 이 비용 함수의 값을 줄여 (오차를 줄임) 적절한 W,b 값을 찾아내는것이 목표 



#### 5) 경사 하강법

경사 하강법 (Gradient Descent) 는 가설함수를 최적화(오차를 줄임)는 방법 중 하나이다.
- 기울기를 활용함
- 미분이 등장
- 경사를 타고 내려가면서 기울기가 0이 되는 지점까지 내려감
- convex function (밥그릇 모양?) 의 형태가 아니면 최소값이 구해지지 않을수도 있다.

### Logistic Regression

#### 1) 로지스틱 회귀

선형 회귀 는 시험 점수를 예측하는 것이라면 로지스틱 회귀는 P/NP를 예측하는 것!

- 혹은 어떤 메일이 스팸인지 아닌지 분류하는 것 
- 이진 분류(Binary Classification)라고도함

#### 2) Sigmoid 함수

선형회귀 처럼 가설 함수를 W*x+b 로 하였을때 나타나는 오류를 방지하기위해 새 함수를 도입 

- sigmoid(x) = 1/1+e^-x  이래요
- 후에 알아보니 -활성화 함수- 라는 용어가 있네요
- 로지스틱 회귀의 가설함수는 h(x) =sigmoid(wx+b)


#### 3) 비용 함수

새 가설 함수을 위한 새로운 비용함수 모델링 (선형회귀의 최소제곱함수를 쓰면 로컬 미니멈에 빠진대요)

- log 함수의 성질을 이용하여 만든것 같네요 
- 똑같이 경사하강 알고리즘으로 최소오차를 구한다.

<hr>

### Softmax Regression

#### 1) 다중 로지스틱 회귀?

단순 - 다중 선형 회귀의 관계와 비슷한 것 같다. 다만 소프트맥스 회귀는 출력값이 여러개임 
- P/NP , Y/N 가 아닌 여러개의 클래스로 분류 됨 
- 마치 A,B,C,F 등급을 나누는것을 예측하는것 


#### 2) Softmax 함수

여러개의 sigmoid 함수를 거치고 가장 높은 값이 해당 클래스라고 할때

- 모든 출력값을 더했을때 1이 되도록 
- 모든 출력값은 0 ~ 1 사이 이게

여러 출력값을 확률로 만드는 함수

#### 3) 비용함수

Cross Entropy ? 를 이용한댄다 . 

- Cross Entropy 에서 losgistic regression 의 cost function 을 유도 할 수 있다고 함 
- 확률과 통계 지식이 필요해 보인다.. 이런것으로 하는구나 정도로만 알아놨다.

### 학습 관련

#### 1) Learning rate 

cost function 을 최소화 할 때 사용하는 Gradient Descent 라는 알고리즘에서

경사를 타고 내려가는 정도? 를 조절 하기 위해 Learning rate 라는 값을 곱하였다.

- 학습 데이터 마다 적절한 Learning rate 가 존재함 
- 너무 크면 발산해버리고
- 너무 작으면 로컬 미니멈 에 빠지거나 가중치 업데이트가 되지않음

#### 2) Data preprocessing

데이터 전처리는 중요하다. 데이터의 분포에 따라 학습이 잘 될 수도 안 될수도 있다.

- 누락값을 처리하고 쓸모없는 데이터는 없애거나
- 데이터를 정규화를 시킨다거나 
- 통계적 방법으로 데이터를 정제해야 된다고 한다.

#### 3) Trainig Set / Testing Set

모든 데이터를 학습 시키는데 쓰지 말자

- 마치 모의고사 처럼 테스트 셋을 남겨놓자
- 트레이닝 셋에 맞춰져버린 모델을 만들면 좀 그렇다
- 10중 3개정도 남겨놓자 ?
